兼容导出 - export =

兼容导出是我自己给它取的名字，这种方式的导出对应了 CommonJS 中的：

在 TS 中，我们有三种方式导入，其中第三种是官方所推荐的方式：

  • const variableName = require("moduleName") 和 const variableName = require("modueltName").propertyName

  • import {propertyName, ...} from "moduleName" 和 import * as variableName from "moduleName"

  • import variableName = require("moduleName") 和 import variableName = moduleName.propertyName

由于第三种是官方推荐的方式，那就默认第三种！

小结

export = 和 import ... require 是 TypeScript 为了兼容 CommonJS 和 AMD 规范二存在的语法，所以其实并不常用。在绝大多数情况下，还是建议用 named export。
